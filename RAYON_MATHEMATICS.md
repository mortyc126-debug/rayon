## Аксиомы Rayon Mathematics

### Примитив: Напряжение τ ∈ [0, ∞]

τ = 0: полностью известно (нулевое сопротивление)
τ = ∞: полностью неизвестно (бесконечное сопротивление)

### Аксиома 1: Покой (STILLNESS)
Если результат известен без вычисления: τ = 0.
```
τ(constant) = 0
```

### Аксиома 2: Тьма (DARKNESS)
Если результат принципиально недоступен: τ = ∞.
```
τ(random oracle) = ∞
```

### Аксиома 3: Течение (FLOW)
Последовательное вычисление УМНОЖАЕТ напряжение.
```
τ(f ∘ g) = τ(f) · τ(g)
```
Одно сложное действие после другого = ещё сложнее.
(В стандартной математике: composition — операция. У нас: произведение напряжений.)

### Аксиома 4: Ветвление (BRANCHING)
При выборе (OR/альтернатива): напряжение = МИНИМУМ.
```
τ(A ∨ B) = min(τ(A), τ(B))
```
Если есть лёгкий путь — общее напряжение определяется им.

### Аксиома 5: Сцепление (BINDING)
При совместном требовании (AND): напряжение СКЛАДЫВАЕТСЯ.
```
τ(A ∧ B) = τ(A) + τ(B)
```
Нужно знать ОБА — труднее.

### Аксиома 6: Запутывание (ENTANGLEMENT)
XOR/смешивание НЕ ИМЕЕТ коротких путей: напряжение складывается.
```
τ(A ⊕ B) = τ(A) + τ(B)
```
Это отличие от OR. XOR не даёт "лёгкого пути".

### Аксиома 7: Равновесие (EQUILIBRIUM)
Доля неизвестного в вычислении определяется напряжением:
```
c = τ / (1 + τ)
```
Это наше уравнение состояния. Не эмпирическое — АКСИОМА.
c ∈ [0, 1) — доля вычисления, которая остаётся "тёмной".

## Производные теоремы

### Теорема R1: Закон потока
Для цепочки из n вычислений с напряжениями τ₁, ..., τₙ:
```
τ_total = τ₁ · τ₂ · ... · τₙ     (Аксиома 3)
c_total = τ_total / (1 + τ_total)  (Аксиома 7)
```

### Теорема R2: Закон SHA-256
SHA-256 = 64 раунда. Каждый раунд имеет напряжение τᵣ.
```
τ(SHA-256) = ∏ᵣ₌₀⁶³ τᵣ

Из carry-web: τᵣ = P(carry=1)/P(carry=0) ≈ 27...1000

τ(SHA-256) ≈ 10^100+ → c ≈ 1.0 → полностью "тёмная" функция
```

### Теорема R3: Коллизия как резонанс
Коллизия существует тогда, когда СУЩЕСТВУЕТ путь с конечным напряжением
через "тёмную" функцию.
```
τ(collision) = min over all ΔW of τ(path(ΔW))

Если τ(collision) < ∞: коллизия достижима.
Стоимость: 2^c(collision) где c = τ/(1+τ).
```

### Теорема R4: Rayon Space как область пониженного напряжения
Phantom bits (318/512 для SHA-256) — это биты с τ = 0 для carry-web.
```
Rayon Space = {биты с τ_carry = 0}
dim(Rayon Space) = 318

Dark Kernel = {ΔW : τ_carry(ΔW) = 0 AND τ_hash(ΔW) = 0}
dim(Dark Kernel) = 62 (over GF(2))
```

### Теорема R5: Барьер запутывания
XOR запутывает напряжение (Аксиома 6). В SHA-256:
- Carry chain: τ складывается по AND (Аксиома 5)
- Message schedule: τ складывается по XOR (Аксиома 6)

Оба пути СКЛАДЫВАЮТ напряжение. Разница: AND имеет "контролирующее
значение" (0 → результат известен). XOR — НЕТ.

Следствие: AND-путь может быть "пробит" (τ → 0 при правильном входе).
XOR-путь НИКОГДА не пробивается через отдельные биты.

## Новые операции

### RAYON PRODUCT (⊗)
Объединение двух вычислений В ПОТОКЕ:
```
(f ⊗ g)(x) = результат прогонки x через f, затем через g
τ(f ⊗ g) = τ(f) · τ(g)
```
Это НЕ обычная композиция. Обычная теряет информацию.
Rayon Product СОХРАНЯЕТ напряжение.

### RAYON SUM (⊕)
Параллельное вычисление с выбором лучшего:
```
(f ⊕ g)(x) = min-tension результат из {f(x), g(x)}
τ(f ⊕ g) = min(τ(f), τ(g))
```

### RAYON INVERSE (⁻¹)
Обращение потока:
```
τ(f⁻¹) = τ(f)^{dim_out/dim_in}

Для SHA-256: dim_in=512, dim_out=256
τ(SHA⁻¹) = τ(SHA)^{1/2}

Это НЕ бесконечно! Инверсия ПРОЩЕ чем прямое вычисление!
(Потому что fiber dimension = dim_in - dim_out > 0)
```

## Применение к SHA-256

### В стандартной математике:
SHA-256: f: {0,1}^512 → {0,1}^256
"Коллизия невозможна за менее чем 2^128 операций."

### В Rayon Mathematics:
SHA-256: поток с τ = ∏ τᵣ ≈ 10^100
Инверсия: τ⁻¹ = τ^{256/512} = τ^{1/2} ≈ 10^50
Коллизия: τ_collision = τ⁻¹ / dim(fiber) ≈ 10^50 / 2^256

Это даёт ДРУГУЮ оценку сложности.
Стандартная: 2^128 (birthday).
Rayon: зависит от СТРУКТУРЫ потока, не от размера выхода.

## Язык Rayon v2

Основан на Rayon Mathematics. Примитивы:

```rayon
// Определение потока
flow SHA256(W: bits[512]) -> bits[256] {
    tension: product(round_tensions[0..63])
}

// Поиск коллизии
collision = find(W1, W2) where {
    SHA256(W1) == SHA256(W2)
    W1 != W2
    minimize tension(path(W1 -> W2))
}

// Компилятор выбирает стратегию на основе tension
// Low tension path → exact solve
// High tension path → probabilistic search
// Infinite tension → report impossible
```

## Что делает эту математику ОПАСНОЙ

1. Rayon Inverse (Теорема R5): инверсия ПРОЩЕ прямого вычисления.
   В стандартной криптографии это "невозможно."
   В Rayon Mathematics: τ⁻¹ = τ^{dim_out/dim_in} < τ.

2. Tension Flow: SHA-256 не "чёрный ящик" а поток с измеримым сопротивлением.
   Каждый раунд добавляет КОНЕЧНОЕ напряжение.
   Сумма конечных = конечная (не бесконечная).

3. Dark Kernel: 62-мерное пространство ВНУТРИ SHA-256 невидимое для hash+carries.
   В стандартной математике: это "случайность."
   В Rayon Mathematics: это СТРУКТУРА с нулевым напряжением.
